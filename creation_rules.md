# プロジェクト制作ルール (量産化ガイド)

このドキュメントは、各ゲーム紹介ページの制作を標準化し、効率的に量産するためのルールを定めたものです。

## 1. ディレクトリ構成
各ゲームページは以下の構成で作成してください。

```text
[プロジェクトルート] /
  ├─ [プロジェクト名] (例: GFF2026) /
  │   └─ [チーム名] (例: Team1_1) /
  │       ├─ index.html
  │       ├─ style.css
  │       ├─ app.js
  │       ├─ [ゲーム動画].mp4
  │       └─ [企画書].pdf
  └─ creation_rules.md (このファイル)
```

## 2. アセットの命名規則
HTML内の書き換えを最小限にするため、以下の名称を推奨します（またはHTML内のソース指定を変更してください）。

- **背景動画**: `background.mp4` (推奨)
- **企画書PDF**: `proposal.pdf` (推奨)

## 3. HTMLのカスタマイズ箇所 (index.html)
量産時に以下の箇所を検索・置換してください。

1.  `<title>`: ゲーム名に合わせて変更。
2.  `<h1 class="glitch-text">`: ゲームタイトルを表示（`data-text`属性も同様に変更）。
3.  `<p class="subtitle">`: プロジェクト部門名（例：GFF2025 ゲームソフト部門）。
4.  `<div class="tag">`: プロジェクトIDやチーム名。
5.  `<iframe src="...">`: PDFのファイル名を指定（`#toolbar=0`などは維持）。
6.  `<video id="large-video">`: 拡大表示用動画のファイル名を指定。

## 4. デザインのカスタマイズ (style.css)
ゲームの雰囲気に合わせ、`:root`のカラー変数を変更してください。

```css
:root {
    --accent-color: #00ff41; /* メインアクセント (ネオン系推奨) */
    --accent-color-alt: #ff0055; /* サブアクセント (Closeボタン等) */
}
```

## 5. 新規ページの作成手順
1.  既存の完成済みフォルダ（例: `Team1_1`）を丸ごとコピーする。
2.  新しい動画とPDFを配置する。
3.  `index.html` 内のタイトル、ファイルパス、タグ情報を書き換える。
4.  `style.css` でアクセントカラーを調整する。
5.  ブラウザで開き、以下の動作を確認する：
    -   動画背景が再生されるか。
    -   「企画書を見る」で正しいPDFが開くか。
    -   「動画を見る」で拡大動画が再生され、背景が止まるか。
